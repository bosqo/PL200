## Exam Structure

### Question Types

The PL-200 exam includes multiple question formats to assess different skill levels:

1. **Multiple Choice**
   - Select one or more correct answers from a list
   - Tests knowledge recall and understanding
   - May have one correct answer or multiple correct answers

2. **Hotspot**
   - Click on specific areas of a diagram or image to answer
   - Tests ability to identify components in visual representations
   - Common for architecture and configuration questions

3. **Drag and Drop**
   - Match items, order steps, or categorize components
   - Tests understanding of sequences and relationships
   - Example: Order configuration steps or match concepts to definitions

4. **Case Studies**
   - Analyze scenarios and answer multiple related questions
   - Tests ability to apply knowledge to real-world situations
   - Usually 2-5 questions per case study

5. **Build List**
   - Arrange items in the correct sequence
   - Tests understanding of proper order of operations
   - Similar to drag and drop but specifically for sequencing

6. **Active Screen**
   - Interact with simulated Power Platform interfaces
   - Tests hands-on knowledge of UI navigation
   - Example: Clicking buttons or entering data in a simulated form

## Exam Coverage Areas

Based on the official Microsoft study guide (updated January 2024), the exam is weighted as follows:

### 1. Configure Microsoft Dataverse (25-30%)
**Key Topics:**
- Manage environments
- Configure security settings (roles, privileges, field-level security)
- Create and manage tables (entities) and columns
- Create and manage relationships and cascade rules
- Implement business logic (business rules, calculated columns, rollup columns)
- Configure alternate keys and duplicate detection
- Manage teams and access teams
- Entity ownership types

**Deep-Dive Resource:** [../learning-material/Dataverse-DeepDive.md](../learning-material/Dataverse-DeepDive.md)

### 2. Create Apps Using Power Apps (25-30%)
**Key Topics:**
- Create model-driven apps (forms, views, charts, dashboards)
- Create canvas apps (responsive design, data sources, formulas)
- Configure forms and views
- Implement Power Pages
- Component libraries and reusable components
- Data connections and delegation
- Gallery controls and form controls

**Deep-Dive Resources:**
- [../learning-material/ModelDrivenApps-DeepDive.md](../learning-material/ModelDrivenApps-DeepDive.md)
- [../learning-material/CanvasApps-DeepDive.md](../learning-material/CanvasApps-DeepDive.md)
- [../learning-material/PowerPages-DeepDive.md](../learning-material/PowerPages-DeepDive.md)

### 3. Create and Manage Logic and Process Automation (25-30%)
**Key Topics:**
- Create cloud flows (automated, instant, scheduled)
- Create business process flows with branching
- Implement triggers and actions in Power Automate
- Configure process automation and workflow triggers
- Error handling and retry policies
- Desktop flows (RPA) - attended vs unattended mode
- Integration with other systems

**Deep-Dive Resources:**
- [../learning-material/PowerAutomate-DeepDive.md](../learning-material/PowerAutomate-DeepDive.md)
- [../learning-material/BPF-DeepDive.md](../learning-material/BPF-DeepDive.md)
- [../learning-material/Integration-DeepDive.md](../learning-material/Integration-DeepDive.md)

### 4. Manage Environments (15-20%)
**Key Topics:**
- Configure environment settings
- Manage data policies and DLP (Data Loss Prevention)
- Implement environment strategies (development, test, production)
- Monitor and troubleshoot environments
- Manage capacity and resources
- Solution management and ALM (Application Lifecycle Management)

## Exam Day Tips

### Before the Exam

1. **Get Adequate Sleep**
   - Don't cram the night before
   - Well-rested mind performs better
   - Aim for 7-9 hours of sleep

2. **Prepare Your Environment**
   - Clear desk of unnecessary items
   - Close all unnecessary programs
   - Ensure good lighting
   - Have water nearby (if online, check proctor policy)

3. **Arrive Early**
   - For test center: Arrive 15 minutes early
   - For online: Log in 10 minutes early
   - Complete identity verification early

4. **Review Key Concepts**
   - Light review of main topics (don't cram new material)
   - Review the exam objectives one more time
   - Focus on areas of weakness

### During the Exam

1. **Read Questions Carefully**
   - Pay attention to keywords like "EXCEPT," "NOT," "least privilege," "minimum permissions"
   - Underline key requirements in your mind
   - Don't rush through the question reading

2. **Manage Your Time**
   - Aim for 2-3 minutes per question maximum
   - Don't spend too long on one question
   - Mark difficult questions for review if time allows
   - For 40-60 questions in 120 minutes: ~2 minutes per question

3. **Answer Strategy**
   - Answer questions you know first to build confidence
   - For multiple choice: Eliminate obviously wrong answers
   - For case studies: Read the scenario once, then answer all related questions
   - Watch for version numbers (classic workflows vs cloud flows, CDS vs Dataverse)

4. **Use Process of Elimination**
   - Remove answers that contradict each other
   - Watch for "trick" answers that are partially correct
   - Remember Microsoft recommends modern solutions (cloud flows over classic workflows)

5. **Answer All Questions**
   - Don't leave questions blank
   - Even a guess is better than no answer
   - You can review marked questions if time permits

### Common Exam Question Keywords

Watch for these terms that change question meaning:

| Keyword | Meaning |
|---------|---------|
| **EXCEPT** | All options are correct EXCEPT... |
| **NOT** | This is not the correct answer |
| **Least privilege** | Minimum permissions needed |
| **Best practice** | Recommended approach |
| **Must** | Required, non-negotiable |
| **Should** | Recommended, best practice |
| **Can** | Possible, not necessarily best |
| **All of the above** | Multiple answers are correct |
| **None of the above** | All options are incorrect |

### Things to Remember

1. **Version Matters**
   - Classic Workflows are deprecated ‚Üí Use Cloud Flows
   - CDS (Common Data Service) is now called Dataverse
   - Update your knowledge to current terminology

2. **Security Questions**
   - "Least privilege" means minimum permissions needed
   - Understand privilege levels: None, User, Business Unit, Organization
   - Know the difference between role-based and record-based security

3. **Architecture Questions**
   - Focus on the "why" not just "how"
   - Understand business requirements first
   - Think about scalability and maintainability

4. **Multiple Answer Questions**
   - Read all options before deciding
   - "Select all that apply" requires identifying ALL correct answers
   - Partially correct is not correct

## FAQ

### Q: Can I bring notes or reference materials to the exam?
**A:** No. For online proctored exams, no reference materials are allowed. Test centers also prohibit external materials.

### Q: Can I retake the exam if I fail?
**A:** Yes. There is typically no waiting period. You can retake immediately (may have a 24-hour cooldown at some centers).

### Q: How long is my certification valid?
**A:** PL-200 certification is valid for one year. You must pass the renewal assessment annually to maintain certification.

### Q: Can I use a calculator during the exam?
**A:** No. No external devices including calculators are permitted. Use on-screen calculator if needed (rarely required).

### Q: What happens if I have technical issues during an online exam?
**A:** Contact the proctor immediately. They can pause the exam and troubleshoot. The time continues, so technical issues may impact your available time.

### Q: What score do I need to pass?
**A:** 700 out of 1000. This is not a percentage-based score - it's a scaled score.
